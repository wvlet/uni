# Base62 URL Encoding & NanoId Generator

## Goal
Add three utilities to `wvlet.uni.util`:
1. **Base62** — `0-9A-Za-z` alphabet (62 chars), URL-safe, sort-order preserving. 128 bits → 22 chars.
2. **UUIDv7 Base62** — `toBase62` instance method, `UUIDv7.fromBase62(s)` factory method.
3. **NanoId** — Default 21 chars, `A-Za-z0-9_-` (64 chars). SecureRandom + rejection sampling (bitmask).

## Design

### Base62
- Alphabet: `0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz`
- Sort-order preserving: lexicographic order of encoded strings matches numeric order
- `encode128bits(hi: Long, low: Long): String` → 22-char string
- `decode128bits(s: String): (Long, Long)` → pair of Longs
- Use BigInteger arithmetic for correct 128-bit → base62 conversion

### UUIDv7 Base62
- `toBase62: String` on UUIDv7 class delegates to `Base62.encode128bits(msb, lsb)`
- `UUIDv7.fromBase62(s: String): UUIDv7` decodes and validates version/variant

### NanoId
- `NanoId.generate(): String` — 21 chars, default alphabet `A-Za-z0-9_-`
- `NanoId.generate(size: Int): String`
- `NanoId.generate(alphabet: String, size: Int): String`
- Uses SecureRandom with bitmask rejection sampling to avoid modulo bias
